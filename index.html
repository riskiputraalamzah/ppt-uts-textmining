Bab 1: Pendahuluan

Latar Belakang

Pertumbuhan pesat e-commerce menghasilkan data ulasan yang berharga namun tidak terstruktur (banyak slang, singkatan, typo).

Tujuan Project

  • Implementasi API Scraping (Otomatisasi pengambilan data).
  • Preprocessing Lengkap (Normalisasi bahasa gaul & penanganan negasi).

Mengapa Tokopedia?

Platform terbesar dengan variasi bahasa ulasan yang sangat beragam (Formal vs Alay).

Sumber Data

  • Produk: Clinelle Twin Pack Moisture Glow.
  • Jumlah Data: > 500 Ulasan.
  • Alasan: Data cukup banyak dan mengandung sentimen variatif.

Metode API Scraping

Kami tidak menggunakan parsing HTML biasa, melainkan Reverse Engineering API GraphQL.


Tantangan: Error 403 (Blokir)

Server Tokopedia memblokir bot otomatis.

Solusi: Session Hijacking

Mengambil Cookies dan Headers asli dari browser (Chrome DevTools) lalu menyematkannya ke script Python.

Implementasi Python

Script scraper dijalankan di Google Colab menggunakan library requests.

  • Payload GraphQL:

    Meminta data spesifik (rating, text, user).

  • Filtering:

    Menghapus ulasan kosong atau terlalu pendek (< 3 kata).

  • Hasil: 622 Data berhasil diambil.

Bab 3: Text Preprocessing

Tahapan Pembersihan

  • 1. Case Folding:

    Mengubah ke huruf kecil.

  • 2. Cleaning:

    Hapus simbol, emoji, dan angka.

  • 3. Normalization:

    Menggunakan Kamus Alay untuk membakukan kata (cth: "bgt" -> "sangat").

  • 4. Stemming:

    Mengubah ke kata dasar (Sastrawi).

Fitur Unggulan

Kami memodifikasi daftar Stopwords agar TIDAK MENGHAPUS kata negasi seperti "tidak" dan "jangan".

Alasan: Agar kalimat "tidak iritasi" maknanya tidak berubah menjadi "iritasi".

Implementasi Modular

Kode preprocessing kami pisahkan ke dalam file modul agar rapi dan mudah dikelola.


Komponen Modul:

  • Load Kamus Alay (GitHub).
  • Load Sastrawi Stemmer.
  • Custom Stopword List.
